以下の手順に従って、指定されたMarkdownファイルをZenn記事形式に変換してください。

## 対象ファイル

$ARGUMENTS

## 変換手順

### 1. ファイルの読み込み

引数として渡されたMarkdownファイルを読み込んでください。引数が空の場合は、ユーザーに変換したいMarkdownファイルのパスを尋ねてください。

### 2. 内容の分析

読み込んだMarkdownの内容を分析し、以下を判断してください：

- **title**: 最初のH1見出し（`# 〜`）からタイトルを抽出する。H1がなければファイル名をタイトルとして使用する
- **type**: 内容が技術的な解説・手順・コードを含む場合は `"tech"`、意見・アイデア・考察の場合は `"idea"`
- **topics**: 記事の内容から関連するトピックタグを最大5つ提案する（例: `["Python", "AWS", "Docker"]`）
- **emoji**: 内容にあった絵文字を1つ選ぶ（例: 技術系なら🔧、クラウドなら☁️、Pythonなら🐍）
- **published**: デフォルトは `false`（下書き）

### 3. フロントマターの生成

以下の形式のZennフロントマターを生成してください：

```yaml
---
title: "抽出したタイトル"
emoji: "選んだ絵文字"
type: "tech"
topics: ["topic1", "topic2"]
published: false
---
```

### 4. 本文の変換

以下のルールに従って本文を変換してください：

- 最初のH1見出し（`# タイトル`）は、フロントマターの `title` に移動したためMarkdown本文からは**削除する**
- 画像参照 (`![alt](path)`) はそのまま維持する
- コードブロック（` ``` `）はそのまま維持する
- その他のMarkdown記法はZenn互換のためそのまま維持する

### 5. 出力ファイルの作成

- `npm run add` を実行して新しい記事ファイルを作成する
- 作成されたファイルのパスを出力から読み取る
- そのファイルを開き、生成したフロントマターと変換後の本文で上書きする

### 6. 変換結果の報告

変換完了後に以下を報告してください：

- 作成したファイルのパス
- 設定したフロントマター（title, emoji, type, topics, published）
- 変換のポイントや注意事項があれば補足する

### 7. プレビューの起動

- `npm run dev` をバックグラウンドで実行してプレビューサーバーを起動する
- ユーザーに `http://localhost:8000/articles/{作成したファイル名（拡張子なし）}` をブラウザで開くよう案内する

## 注意事項

- `published: false` がデフォルトです。公開する場合はユーザーに確認してから `true` に変更してください
- topicsはZennの既存タグを使用することを推奨します（大文字・小文字、表記は既存のものに合わせる）
- ファイル名に使える文字は半角英数字・ハイフン・アンダースコアのみで、12〜50文字にしてください
